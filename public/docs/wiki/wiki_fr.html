Configurer Coctohug sur le système d'exploitation Linux

  • Installer Docker + Docker-Compose
  • Aller sur le site https://www.coctohug.xyz, puis entrez tous les champs de formulaire nécessaires et téléchargez les fichiers zippés docker-compose générés
  • Décompressez les dossiers téléchargés et copiez-les dans votre répertoire de travail. Veuillez également supprimer ces dossiers de fourches blockchain indésirables pour éviter le cas d'ordinateur à court de ressources
  • Installez les fourches blockchain par des scripts similaires :
    ./ccm.sh start flora
./ccm.sh start flax
./ccm.sh start hddcoin
./ccm.sh start chia
...
  • Ouvrez le navigateur et accédez à WebUI avec l'url http://localhost:12630/
  • Remarque 1 : ne démarrez pas plus de 5 forks blockchain en même temps, car les forks chia blockchain consomment vraiment du processeur lors de la première synchronisation des nœuds.
  • Remarque 2 : environ 1,8 G de RAM est requis pour chaque fourche blockchain, alors choisissez des fourches blockchain en fonction de la mémoire de votre ordinateur.
  • Remarque 3 : en cas de problème, vous devrez peut-être redémarrer tous les forks de la blockchain :
    ./ccm.sh restart flora
./ccm.sh restart flax
./ccm.sh restart hddcoin
./ccm.sh restart chia
...
  • Remarque 4: Vous devrez peut-être ajouter des ports autorisés de 12630 à 12700 sur les paramètres de votre pare-feu système
  • Remarque 5: N'oubliez pas de déplacer ces dossiers de fourches blockchain indésirables hors du dossier coctohug. Sinon, ils seront démarrés en même temps que vous opérez sur tous les forks de blockchain fonctionnels, ce qui peut entraîner une pénurie de ressources sur votre ordinateur. ``` ./ccm.sh start all ./ccm.sh restart all ./ccm.sh stop all ./ccm.sh upgrade all ...

Mot de passe

  • Vous serez invité à définir un mot de passe pour les paramètres sécurisés, lorsque vous accéderez pour la première fois à WebUI avec l'url http://localhost:12630/
  • Les paramètres sécurisés incluent : Cold Wallet, Coin Transfer, Redémarrer les fourches Blockchain, Harvester...

 

Clés

  • Vous serez invité à saisir les 24 mots mnémoniques lorsque vous accéderez pour la première fois à WebUI avec l'url - ceci est principalement à des fins agricoles
  • Ou vous pouvez générer une clé totalement nouvelle - c'est principalement à des fins de portefeuille froid

 

Exploitation minière

  • Une fois la synchronisation des nœuds terminée, l'agriculture devrait démarrer automatiquement
  • Ensuite, vous pourrez surveiller l'état de l'exploitation minière de différentes manières

 

Surveillance

  • L'onglet Résumé peut afficher : le total des pièces extraites, le solde du compte, le nombre de parcelles, la taille de la parcelle, la taille de l'espace net et le temps prévu pour gagner
  • chaque panneau de fourche blockchain dans l'onglet Résumé aura un fond vert clair s'il fonctionne bien, sinon ce sera un fond jaune clair
  • Vous pouvez également examiner les détails en utilisant d'autres onglets

 

Récupération NFT

  • L'onglet Pièces reçues a un lien : Réclamez vos récompenses NFT 7/8

 

Blocs trouvés / Pièces reçues

  • Liste des pièces totales reçues

 

Avertissements miniers

  • Liste des problèmes de point de signature possibles
  • Liste des problèmes de recherche de disque possibles
  • Liste des problèmes possibles de diminution du nombre de parcelles

Synchronisation des nœuds

  • Le nœud se synchronisera automatiquement par défaut
  • Vous pouvez accéder à l'onglet Connexions ou Blockchains pour examiner l'état détaillé
  • Pour accélérer la synchronisation de vos nœuds : Coctohug site fournit des liens vers des fichiers de base de données - développé et sponsorisé par : Maize fork
    1. Téléchargez les fichiers : blockchain_v1_mainnet.sqlite et peer_table_node.sqlite
2. Arrêtez la fourche blockchain par un script de terminal similaire (à l&#39;intérieur de chaque dossier):
  Windows: .\ccm.ps1 stop all
  Linux / Mac: ./ccm.sh stop all
3. Copiez ces fichiers dans le dossier de la fourche blockchain de la manière suivante :
  Windows: /c:/users/username/.coctohug-covid/covid/mainnet/db/
  Linux / Mac: /home/username/.coctohug-flora/flora/mainnet/db/
4. Démarrez blockchain fork par un script de terminal similaire (à l&#39;intérieur de chaque dossier):
   Windows: .\ccm.ps1 restart all
   Linux / Mac: ./ccm.sh restart all
Notes 1 : votre portefeuille a besoin d&#39;une synchronisation individuelle dans le programme fork
Notes 2 : La connexion de la base de données téléchargée à votre fork blockchain peut prendre jusqu&#39;à 30 minutes !

  • Répertorier toutes les connexions de nœuds

Gestion des portefeuilles

  • Lister le statut du portefeuille et le solde du compte
  • Vous pouvez également transférer des pièces dans l'onglet portefeuille

 

Gestion des mains

  • Vous pouvez retirer une main lorsque vous ne prévoyez plus de la cultiver

 

Transférer des pièces

  • Allez dans l'onglet portefeuille et entrez votre mot de passe sécurisé pour transférer des pièces

 

Portefeuille froid

  • Accédez à l'onglet Paramètres avec votre mot de passe sécurisé
  • Exporter un compte Cold Wallet
    1. Préparer une nouvelle machine (différente avec la machine minière)
2. Visite https://www.coctohug.xyz et cliquez sur Wallet Mode pour générer des dossiers docker-compose
3. Configurer les dossiers docker-compose [Configurer Coctohug sur le système d&#39;exploitation Linux]
4. Sur l&#39;écran de lancement de l&#39;interface Web, générez une nouvelle clé cette fois
5. Attendez quelques minutes que les forks de la blockchain redémarrent
6. Visite http://localhost:12630/, et allez dans les paramètres - onglet portefeuille froid pour exporter les adresses du portefeuille froid
7. Confirmez que chaque adresse est correcte manuellement en comparant le fichier téléchargé avec les informations affichées dans l&#39;onglet Clés
8. Obtenez 24 mots mnémotechniques par un script de terminal similaire
  docker exec -it coctohug-flora flora keys show --show-mnemonic-seed
  docker exec -it coctohug-covid covid keys show --show-mnemonic-seed
  docker exec -it coctohug-lucky lucky keys show --show-mnemonic-seed
  • Importer un compte Cold Wallet
    1. Sur votre machine de minage, visitez les paramètres - onglet portefeuille froid
2. Importez le fichier json du portefeuille froid téléchargé précédent (recommandé sur une autre machine)
3. Attendez quelques minutes et accédez à l&#39;onglet portefeuille pour voir si l&#39;adresse du portefeuille froid est mise à jour ou non
4. Remarque 1 : nous vous recommandons de sauvegarder vos configurations de portefeuille avant d&#39;importer
5. Remarque 2 : seuls les forks de blockchain fonctionnels peuvent importer un cold wallet. Veuillez vérifier s&#39;il y a des fourches blockchain arrêtées ou non avant de faire cela. Vous pouvez sûrement aussi importer à nouveau plus tard lorsqu&#39;ils sont redémarrés
6. Remarque 3 : en cas de problème, vous devrez peut-être redémarrer tous les forks de la blockchain :
  Windows: .\ccm.ps1 restart all
  Linux / Mac: ./ccm.sh restart all

 

Mots mnémotechniques sécurisés avec phrase de passe 24

  • coctohug-passphrase peut être utilisé pour sécuriser vos 24 mots mnémotechniques
  • Il crypte vos clés avec un mot de passe que vous seul connaissez
  • Vous pouvez le déchiffrer avec le mot de passe lors de l'ajout de nouveaux forks blockchain

 

Fourches Blockchain

  • Vérifiez coctohug github tous les quelques jours
  • De nouveaux forks blockchain seront pris en charge très bientôt

 

Moissonneuse

  • Visite https://www.coctohug.xyz et cliquez sur Harvester Mode pour générer des dossiers docker-compose
  • Visite http://localhost:12630/ et cliquez sur Autoriser la moissonneuse pour établir le lien entre le contrôleur et la moissonneuse
  • Configurer les dossiers docker-compose Configurer Coctohug sur le système d'exploitation Linux
  • Quelques minutes plus tard, vous pourrez voir la moissonneuse sur l'onglet mains de WebUI

 

Améliorer

  • Cela nécessite à nouveau les 24 mots mnémoniques. Veuillez les mettre dans le fichier /home/user/.coctohug/mnc.txt. Une fois tous les nouveaux conteneurs Docker démarrés, vous pouvez ensuite vider le fichier mnc.txt pour améliorer votre niveau de sécurité
  • Script d'une ligne
    Windows: .\ccm.ps1 upgrade all; .\ccm.ps1 start all
Linux / Mac: ./ccm.sh upgrade all && ./ccm.sh start all
  • Vous pouvez également exécuter les scripts ci-dessus étape par étape
    .\ccm.ps1 upgrade all
.\ccm.ps1 start all
  • En cas de problème de base de données incompatible, vous pouvez supprimer le fichier de base de données existant avant d'exécuter le script de démarrage docker-compose en
    rm ~/.coctohug-web/db/coctohug.sqlite

 

Langue locale

  • En haut à droite de l'interface Web, choisissez la langue que vous préférez
  • Vous pouvez passer à d'autres langues plus tard si vous le souhaitez

 

Exigences matérielles

  • Une fois synchronisés, les processeurs Intel® Core™ i7 de 10e génération devraient suffire à cultiver plus de 50 forks blockchain
  • Cependant, pour l'étape de synchronisation initiale des nœuds, cela consomme vraiment du processeur. Nous vous recommandons donc de configurer 5 forks blockchain par groupe et de commencer groupe par groupe
  • La mémoire nécessaire est égale à : le nombre de fourches blockchain minières multiplie 1,8 G de RAM
  • Le disque normal devrait convenir à plus de 50 forks blockchain
